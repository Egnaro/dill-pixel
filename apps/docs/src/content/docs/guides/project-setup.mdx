
## Organizing Files and Directories

We recommend setting up your project directory as follows:

<FileTree>

- package.json
- README.md
- tsconfig.json
- vite.config.mjs
- index.html
- assets - configured for [assetpack](https://github.com/pixijs/assetpack) (example below)
  - audio\{m\}
    - music
      - music.wav
    - sfx
      - click.ogg
      - beep.ogg
      - boop.wav
  - ui\{tps\}
    - button.png
    - arrow.png
  - game\{tps\}
    - obstacle.png
    - rocket.png
  - spine\{m\}
    - animation.skel
    - animation.atlas
    - animations.png
- **src**
  - dill-pixel.config.ts - configuration file for dill pixel
  - index.ts
  - index.css
  - bootstrap.ts - where you initialize your game
  - MyApplication.ts - extends a dill pixel Application (optional)
  - scenes - where your scenes are stored,<br/>- automatically registered with dill pixel
    - Splash.ts
    - Intro.ts
    - Game.ts
    - HighScores.ts
  - locales - where you store your locale files
    - en.ts
    - fr.ts
  - gameobjects - where you store your game objects
    - Player.ts
    - Enemy.ts
  - utils - where you store your utility functions
    - utils.ts
  - â€¦

</FileTree>

## Vite Configuration

Dill Pixel uses Vite under the hood to compile your game and export it for both development and production environments.

The `vite.config.mjs` file in the root directory configures the build process. The default Vite configuration can be imported from the `dill-pixel` package, and should be sufficient for most projects.
